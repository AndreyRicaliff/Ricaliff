# 05 — Raciocínio de Engenheiro

O que separa júnior de pleno não é decorar API — é o processo mental: verificar antes de afirmar, debugar por hipótese e refutação, decidir por trade-off nomeado e saber quando parar. Esta trilha destila esse processo em 7 hábitos operacionais, cada um com caso real dos projetos AG e resposta de entrevista pronta.

Ordem sugerida (é uma progressão):

1. `01-verificar-antes-de-afirmar` — a base de tudo: evidência > confiança
2. `02-hipotese-e-refutacao` — debugging científico
3. `03-trade-offs-explicitos` — decisão com critério e ADR de 6 linhas
4. `04-ler-o-erro-de-verdade` — stack trace, camadas, reprodução
5. `05-decompor-problemas` — cortar o problema, paralelizar, bloqueador não para o resto
6. `06-verificar-o-alvo-certo` — o par do 01: provar QUAL alvo foi lido
7. `07-quando-parar` — medir antes de otimizar, good enough explícito

Pré-requisitos: nenhum. Este é o módulo transversal da trilha — leia antes (ou junto com) qualquer trilha técnica; todos os outros módulos assumem estes hábitos.
