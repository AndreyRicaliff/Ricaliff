# 42 — Design de Interface

Trilha de design para dev que constrói UI de verdade: hierarquia, tipografia, cor, espaço, estados, acessibilidade, sistema e crítica. O objetivo não é virar designer — é ter **critério nomeável** para cada decisão visual e saber testá-la com evidência (teste de 5 segundos, passada de teclado, grep de tokens), no espírito de toda a trilha: hipótese antes de opinião, evidência antes de "pronto".

**Ordem sugerida:**
1. `01-hierarquia-visual` — as 4 alavancas que guiam o olho
2. `02-tipografia` — escala, medida, line-height, tabular-nums
3. `03-cor-com-intencao` — HSL, semântica, contraste AA, dark mode
4. `04-espacamento-grid` — escala 4/8, Gestalt, densidade por contexto
5. `05-estados-da-interface` — os 5 estados; vazio é a primeira impressão
6. `06-acessibilidade-minima` — o mínimo profissional não-negociável
7. `07-design-system-pratico` — tokens, variantes, documentar o porquê
8. `08-critica-estruturada` — a síntese: avaliar tela com método

**Pré-requisitos:** trilha `40-frontend` (React/Tailwind básicos); o módulo 06 conversa com `60-seguranca` no hábito de auditar antes de afirmar. O módulo 08 pressupõe os 7 anteriores — é o exame da trilha.
